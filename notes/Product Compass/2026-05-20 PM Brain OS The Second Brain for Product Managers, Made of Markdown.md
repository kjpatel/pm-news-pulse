# PM Brain OS: The Second Brain for Product Managers, Made of Markdown

**Source**: [Product Compass](https://www.productcompass.pm/p/pm-brain-os)
**Author**: Pawel Huryn | **Date**: May 20, 2026

---

## Summary

PM Brain OS is a markdown-based second brain system for product managers that uses Claude to synthesize context from disparate sources (Notion, Linear, Slack, dashboards) while maintaining an auditable, decision-focused knowledge base that resists common AI memory system failures.

## Key Takeaways

- **Build with epistemic rigor** - Tag all claims by type (observation, interpretation, hypothesis, assumption, decision) to preserve signal instead of flattening contradictions into consensus, enabling better PM judgment with an audit trail.
- **Implement weekly maintenance sweeps** - Run regular synthesis passes to compress recurring patterns, flag stale evidence, and surface strategy drift before it compounds, rather than accumulating everything indefinitely.
- **Keep systems grep-able and editable** - Store everything in markdown files within git repos that any editor can open, making the system trustworthy and version-controllable instead of relying on opaque vector databases.
- **Surface tensions without blocking** - Design the system to flag contradictions between stakeholder claims and documented evidence for PM judgment, not as hard gates that prevent decision-making or usage.
- **Resist feature creep toward simplicity** - Avoid taxonomies, agent swarms, and complex ontologies; instead build an opinionated, lightweight system that can be installed and productive in 10 minutes.
