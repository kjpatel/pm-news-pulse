# Your .claude/ Folder Is a Production Agent

## Summary

Your development .claude/ folder configuration is directly deployable as a production agent runtime without translation layers—the same CLAUDE.md, skills, MCPs, and hooks that guide Claude Code development become your production agent's executable format.

## Key Takeaways

- **Deploy your .claude/ folder directly** — The CLAUDE.md, skills, and MCP configurations you iterate on in Claude Code are the exact same files that run in production; no translation layer needed between dev and prod environments.
- **Use the Anthropic Agent SDK for app-embedded agents** — For request-response agents inside your existing application, the SDK is a smaller dependency than full platforms like n8n or CrewAI, with clean separation between app logic (routing, auth, CRUD) and agent reasoning.
- **Leverage hooks for production guardrails** — PreToolUse and PostToolUse hooks provide security (block dangerous operations) and observability (log cost, duration, file reads) without requiring external workflow orchestration.
- **Iterate agent behavior at documentation speed** — Changing agent behavior is editing a markdown file, not a code review or redeploy cycle—PMs can prototype and ship agent features without becoming a bottleneck.
- **Scale agents with sub-agent directories** — Each .claude/agents/<name>/ directory is a self-contained agent with its own CLAUDE.md, skills, and MCPs; adding a second agent is just a new folder with identical structure.
