# Skateboards, Cars, and the Return of Code-First Discovery

**Source**: [Itamar Gilad](https://itamargilad.com/code-first-discovery/)
**Author**: Itamar Gilad | **Date**: Jun 03, 2026

---

## Summary

As AI coding tools make building prototypes faster and cheaper, product teams are reverting to code-first discovery approaches instead of validating ideas through cheaper no-code techniques first, which ultimately wastes resources and slows true product learning.

## Key Takeaways

- **Validate before coding**: Use the AFTER model to test assumptions through cheaper no-code methods (interviews, surveys, fake door tests, data analysis) before investing engineering resources in building code prototypes.
- **Resist the speed trap**: AI coding speed doesn't eliminate the time-intensive bottleneck of actually running user tests (recruiting, scheduling, engagement, analysis), so building fast doesn't mean you'll learn fast.
- **Combat escalation of commitment**: Polished prototypes create psychological attachment to ideas; maintain throwaway mentality by keeping early prototypes rough and being willing to kill ideas that test poorly.
- **Question output-focused metrics**: Stop treating 'working code shipped' as the primary measure of progress; instead measure learning velocity and validated assumptions to avoid building the wrong thing efficiently.
