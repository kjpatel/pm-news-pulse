# How to Ship Your First Pull Request as a PM

**Source**: [Product Growth](https://www.news.aakashg.com/p/pm-guide-ship-production)
**Author**: Aakash Gupta | **Date**: Apr 14, 2026

---

## Summary

PMs can now ship code directly using AI tools like Claude Code, fundamentally changing how product work gets done by enabling faster feedback loops and sharper product decisions while keeping engineering focused on complex architectural problems.

## Key Takeaways

- **Expand your PM toolkit** beyond specs and prioritization to include shipping small code changes (copy, config, UI tweaks) that don't require engineering—this tightens feedback loops and sharpens your product strategy within weeks instead of quarters.
- **Move planning from Google Docs to git-based PLANNING.md files** stored in the same repository as code, enabling both humans and AI to reference specs directly during implementation and creating version-controlled accountability.
- **Encode PM judgment in CLAUDE.md** as a persistent instruction file that tells Claude Code your product context, coding standards, and boundaries for what PMs should touch, so you get consistent output across all sessions.
- **Start shipping today without a terminal** by using GitHub's web interface to create repos and write markdown files—you don't need command-line skills to begin contributing code changes.
- **Earn engineer trust by shipping high-quality PRs** through self-review tools and skill files that catch mistakes before engineers see them, positioning code review as the new bottleneck (not code writing).
