# The PM's Guide to AI Design that isn't Slop

**Source**: [Product Growth](https://www.news.aakashg.com/p/pm-guide-ai-design)
**Author**: Aakash Gupta (featuring Meng To) | **Date**: Jul 09, 2026

---

## Summary

This article is a masterclass on using Codex to create high-quality AI designs that avoid the pitfalls of generic outputs, featuring insights from design thought leader Meng To on proper setup, workflow, and iteration techniques.

## Key Takeaways

- **Build cascading context folders** organized by project with a single parent directory so AI loads relevant information automatically without token waste on irrelevant data.
- **Install the taste skill** (60K+ GitHub stars) and use Codex's screenshot shortcut (cmd+cmd) to provide visual context that's more effective than written descriptions for design iterations.
- **Follow the Plan-Build-Iterate workflow**: use Plan mode before building, invoke the taste skill to generate divergent designs, and allocate 20% of effort to human-guided iterations to eliminate slop.
- **Choose output format strategically**: default to HTML for speed and control, use Figma only when designers need to iterate further, and avoid unnecessary tool switching that creates context loss for AI.
- **Leverage Codex Mobile and mid-flow steering** to check builds from anywhere and interrupt/redirect the AI in real-time rather than waiting for complete outputs before course-correcting.
