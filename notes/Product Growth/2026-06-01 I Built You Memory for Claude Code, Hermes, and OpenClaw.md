# I Built You Memory for Claude Code, Hermes, and OpenClaw

## Summary

The author built three markdown-based memory systems for AI agents (Hermes, OpenClaw, and Claude Code) that combine the ambient capture approach of GBrain with the intentional curation of Karpathy's Second Brain to solve the problem of AI amnesia across sessions.

## Key Takeaways

- **Create ambient memory capture** by pointing your agent to add-gbrain-ambient-memory-to-any-agent.md to automatically file conversational dumps and screenshots into your knowledge base without manual curation.
- **Build deliberate knowledge wikis** using add-a-karpathy-wiki-to-your-os.md for bounded projects where you curate specific sources and let Claude generate interlinked pages with citations and conflict markers.
- **Combine both approaches in Claude Code** by implementing a hybrid memory system that captures ambient information while enforcing intentional curation through hooks and idempotent compilation passes.
- **Use plain markdown and avoid external dependencies** like Telegram or databases—keep memory systems lightweight and integrated directly into your existing OS setup for maximum usability.
- **Add lifecycle markers to distinguish hypotheses from decisions** by implementing conflict detection and source tracking to prevent old assumptions from silently overwriting validated decisions.
