## Summary

An OpenAI PM shares advanced techniques for using Codex, including setting up persistent harness systems with data connectors, permissions models, and reusable skills, along with three key automations that run autonomously to handle Slack triage, dashboard updates, and weekly stakeholder reporting.

## Key Takeaways

- **Set up a harness system** with data connectors, permission levels (read-only autonomy, synthesis drafts, human approval required), and reusable skills to transform Codex from a one-off tool into a persistent productivity system.
- **Build automations that run while you sleep** including daily Slack inbox triage, self-updating dashboards that pull from multiple sources, and auto-drafted weekly stakeholder updates to eliminate time-consuming manual tasks.
- **Replace PRDs with prototypes plus companion docs** that include input/output specs and FAQ-style answers to edge cases, which accelerates feedback cycles and shifts engineer conversations from "what do you want" to "how do we improve it."
- **Create reusable skills for repetitive workflows** like experiment reviews that automatically generate hypotheses, monitor results, and produce postmortems—anyone (PM, analyst, or engineer) can build these, not just engineers.
- **Spec your output before prompting** by clearly defining input sources, expected output format, and audience use case upfront to reduce correction loops and improve Codex-generated prototype quality.
