## Summary

Anthropic engineers are replacing Markdown with HTML in Claude Code workflows to create richer, more engaging visual formats for planning and implementation, leading to better human-AI collaboration and product outcomes.

## Key Takeaways

- **Use HTML instead of Markdown** for AI planning and specs to create interactive, visual mockups that improve engagement and understanding compared to text-based documentation.
- **Build throwaway micro-UIs** in HTML to edit specific parts of your plans and allow users to interact with implementations during the planning phase rather than after completion.
- **Create living design systems** as HTML artifacts that live in your repository and travel with your codebase, making design decisions persistent and accessible across projects.
- **Allocate 99% of AI tokens to planning and interfaces** rather than production code, using HTML specs as artifacts to guide implementation with flexibility while maintaining constraints.
- **Adopt a compute allocator mindset** by treating HTML as interactive communication tools that keep humans in the loop without over-constraining Claude's capabilities.
