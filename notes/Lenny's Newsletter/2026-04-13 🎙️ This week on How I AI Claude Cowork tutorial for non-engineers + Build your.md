## Summary

This episode showcases two practical approaches to AI-powered automation: Yash Tekriwal's custom Slack inbox built with Perplexity Computer that reduces 150 daily notifications to 30 actionable ones, and JJ Englert's Claude Cowork tutorial for non-technical workers to automate their entire daily workflow without coding.

## Key Takeaways

- **Build systems, not tasks**: Use AI to create deterministic tools that run automatically via APIs rather than asking AI to repeatedly perform the same task. Yash's Slack digest uses AI once to build code that categorizes notifications, then relies on structured data processing.
- **Create an anti-to-do list**: Identify repetitive tasks you despise (email spam deletion, Slack triage, meeting notes entry) and dedicate time each week to automating them away. This compounds over time into significant productivity gains.
- **Start with a 'brain' file**: Document your working style, preferences, team context, and communication patterns in a Markdown file that Claude reads before every task. This eliminates constant re-explanation and ensures consistent, personalized outputs.
- **Use multi-model orchestration for complex builds**: Perplexity Computer's automatic model switching (Sonnet for data, Gemini for planning, Opus for complexity) eliminates frustrating single-model bottlenecks and simplifies cloud deployment with pre-authenticated connectors.
- **Personalized micro-software is the future**: Instead of one-size-fits-all SaaS, expect a Cambrian explosion of specialized, affordable tools built on existing platforms. Knowledge workers will increasingly have custom apps optimized for their unique workflows.
